# 📚 读书笔记博客 - 完整使用指南

> 最后更新：2026-01-16

## 🎯 核心功能

### 1. 独立笔记页面
每篇笔记都有自己的专属网页，可以单独分享链接。

### 2. 富文本编辑器
类似 Word 的编辑体验：
- ✏️ 加粗、斜体、下划线
- 📝 标题层级（H1/H2/H3）
- 💬 引用格式（适合摘录原文）
- 📋 有序/无序列表
- 🎨 文字颜色和背景色
- 📐 对齐方式

### 3. 书籍摘录功能 ⭐ 新功能
- 独立的摘录管理区域
- 一条条添加精彩段落
- 自动随机颜色展示
- 响应式卡片布局

### 4. 自动化部署
- 在线编辑自动触发构建
- 本地编辑一键部署
- 无需手动管理 data.js

## 📝 两种编辑方式

### 方式 1：在线编辑（推荐）

**优点**：
- 🌐 随时随地编辑
- ✨ 富文本编辑器
- 🚀 自动部署
- 📱 手机也可用

**使用步骤**：
1. 访问 `https://你的用户名.github.io/admin.html`
2. 首次使用配置 GitHub Token：
   - 用户名：`bwx-0910`
   - 仓库名：`bwx-0910.github.io`
   - Token：[点击生成](https://github.com/settings/tokens/new?scopes=repo&description=Blog%20Editor)
3. 点击"+ 新建笔记"
4. 填写信息：

   - 图标 Emoji

**优点**：
- 💻 离线可用
- 🔧 完全控制
- 📂 文件管理清晰

**使用步骤**：
1. 在 `blog/notes/` 目录创建 `.md` 文件
2. 按照模板格式编写：

3. 运行 `build.bat` 生成页面
4. 运行 `deploy.bat` 部署到 GitHub

## 🚀 部署流程

### 自动部署（在线编辑）
1. 在 admin.html 保存笔记
2. GitHub Actions 自动运行
3. 1-2 分钟后生效
4. 强制刷新查看（Ctrl+F5）

### 手动部署（本地编辑）
```bash
# 方式 1：一键发布
publish.bat

# 方式 2：分步执行
build.bat   # 生成页面
deploy.bat  # 推送到 GitHub
```

### 混合使用（本地+在线）
如果你既在网页编辑，又在本地修改：
```bash
sync-and-publish.bat
```
这个脚本会：
1. 先拉取远程更改
2. 合并本地修改
3. 重新构建
4. 推送部署

## 🎯 常见问题

### Q1: 保存后网页没更新？
**A**: 需要等待 1-2 分钟让 GitHub Pages 构建。然后**强制刷新**（Ctrl+F5 或 Cmd+Shift+R）。

### Q2: 推送冲突怎么办？
**A**: 先拉取再推送：
```bash
git pull origin main
git push origin main
```

### Q3: 摘录颜色是随机的吗？
**A**: 是基于索引的"伪随机"，每次生成相同的笔记，颜色顺序一致。

### Q4: 可以修改颜色方案吗？
**A**: 可以！编辑 `css/note-style.css` 中的 `.quote-color-1` 到 `.quote-color-10`。

### Q5: 富文本编辑器支持图片吗？
**A**: 目前不支持。建议使用图床链接，在 Markdown 中插入。

### Q6: 如何备份笔记？
**A**: 所有笔记在 `blog/notes/` 目录，定期备份此文件夹即可。

## 💡 使用建议

### 1. 笔记命名
- ✅ 使用书名：`《原则》读书笔记.md`
- ✅ 主题归类：`心理学-认知偏差.md`
