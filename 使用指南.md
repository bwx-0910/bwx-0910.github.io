# 📚 完整使用指南

## 🎯 现在的工作流程（超简单！）

### 方式 1：一键发布（最推荐！）

```
1. 写 Markdown 笔记 → notes/*.md
2. 双击 publish.bat
3. 完成！
```

### 方式 2：分步操作

```
1. 写 Markdown 笔记 → notes/*.md
2. 双击 build.bat （转换）
3. 双击 deploy.bat （部署）
4. 完成！
```

## 📝 三种写笔记方式

### 方式 A：笔记生成助手（最快）

```bash
1. 双击 add-note.bat
2. 回答几个问题
3. 自动生成模板
4. 编辑内容
5. 双击 publish.bat
```

### 方式 B：复制模板（灵活）

```bash
1. 复制 notes/笔记模板.md
2. 重命名为书名
3. 编辑内容
4. 双击 publish.bat
```

### 方式 C：从头开始（自由）

```bash
1. 创建 notes/书名.md
2. 添加 Front Matter
3. 写笔记内容
4. 双击 publish.bat
```

## 🎨 Markdown 格式

### 必需的 Front Matter

每个 .md 文件开头必须有：

```markdown
---
title: 《书名》读书笔记
icon: 📖
date: 2026-01-14
category: 分类
tags: 标签1, 标签2, 标签3
excerpt: 简短摘要（100-200字）
---

# 笔记正文内容...
```

**注意**：
- 必须有开头和结尾的 `---`
- 所有字段都必填
- 标签用逗号分隔
- 日期格式：YYYY-MM-DD

### Markdown 语法

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体**
*斜体*

- 无序列表
1. 有序列表

> 引用

`代码`

[链接](URL)
```

## 🔧 脚本说明

### add-note.bat
**功能**：问答式生成笔记模板  
**用法**：双击运行，回答问题

### build.bat
**功能**：将 notes/*.md 转换为 js/data.js  
**用法**：写完笔记后运行

### deploy.bat
**功能**：部署到 GitHub Pages  
**用法**：构建后运行

### publish.bat ⭐
**功能**：一键构建+部署  
**用法**：写完笔记直接运行（最推荐）

## ⚠️ 重要提示

### 不要手动编辑 data.js

`js/data.js` 是自动生成的，手动修改会被覆盖！

如果要修改笔记：
- ✅ 编辑 `notes/*.md` 文件
- ✅ 运行 `build.bat`
- ❌ 不要直接改 `data.js`

### 文件编码

所有 .md 文件必须保存为 **UTF-8 编码**！

### Front Matter 必须正确

如果 Front Matter 格式错误，该笔记会被跳过。

## 🔍 故障排除

### 问题：build.bat 报错 "Python 未安装"

**解决**：
1. 访问 https://www.python.org/downloads/
2. 下载并安装 Python
3. 安装时勾选 "Add Python to PATH"

### 问题：笔记没有显示在网站上

**检查**：
1. Front Matter 格式是否正确？
2. 是否运行了 build.bat？
3. 是否运行了 deploy.bat？
4. 文件是否保存为 UTF-8 编码？

### 问题：中文乱码

**解决**：
- 确保文件保存为 UTF-8 编码
- 在 VS Code 中：右下角选择编码 → UTF-8

### 问题：部署失败

**参考**：查看 deploy.bat 的错误提示，或查看之前的部署指南。
