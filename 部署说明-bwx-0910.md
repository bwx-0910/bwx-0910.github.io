# 🚀 专属部署指南 - bwx-0910

你的博客地址：**https://bwx-0910.github.io**

## ⚡ 快速部署（3 步完成）

### 步骤 1️⃣：在 GitHub 创建仓库

1. 访问：https://github.com/new
2. 填写仓库信息：
   - **Repository name（仓库名）**: `bwx-0910.github.io`（必须完全一致！）
   - **Public**（公开）：必须勾选
   - **不要勾选任何初始化选项**（README、gitignore 都不要勾选）
3. 点击绿色按钮 **"Create repository"**

### 步骤 2️⃣：运行部署脚本

1. 回到电脑，进入 `blog` 文件夹
2. **双击 `deploy.bat`**
3. 等待脚本运行完成

脚本已经预设了你的用户名，无需手动输入！

### 步骤 3️⃣：访问你的博客

等待 **1-2 分钟**后，在浏览器访问：

🌐 **https://bwx-0910.github.io**

恭喜！你的博客上线了！🎉

## 📝 修改内容

1. **打开文件**：`blog/js/data.js`
2. **找到 `notes` 数组**
3. **修改或添加你的笔记**：

```javascript

## 第一部分

内容...

## 第二部分

内容...
    `
}
```

4. **保存文件**
5. **再次双击 `deploy.bat`** 更新网站

## ⚠️ 首次部署可能需要登录

运行 `deploy.bat` 时，如果是第一次推送到 GitHub，可能会：

1. **自动打开浏览器** - 登录 GitHub 并授权
2. **或弹出登录框** - 输入 GitHub 用户名和密码

**注意**：密码处需要使用 Personal Access Token（不是登录密码）

### 如何获取 Token

1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token (classic)"**
3. 勾选 **`repo`** 权限
4. 拉到最下面，点击 **"Generate token"**
5. **复制生成的 token**（只显示一次，记得保存）
6. 在 Git 要求输入密码时，粘贴 token

## 🆘 遇到问题？

### 问题 1：推送失败

**原因**：可能需要登录或网络问题

**解决**：
- 检查网络连接
- 按提示登录 GitHub
- 确认仓库名称正确

### 问题 2：网页显示 404

**原因**：部署还没完成或仓库设置问题

**解决**：
- 等待 2-3 分钟
- 访问 https://github.com/bwx-0910/bwx-0910.github.io/settings/pages
- 检查 Source 是否设置为 `main` 分支

### 问题 3：样式没有加载

**解决**：
- 清除浏览器缓存（Ctrl + Shift + Delete）
- 或强制刷新（Ctrl + F5）
